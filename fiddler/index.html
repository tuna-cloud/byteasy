Http抓包

说明

抓取http接口不需要证书，但是抓取Https接口需要添加系统根证书，详细可参考“证书安装”步骤

Roadmap

  • 拦截请求在定义Response时，支持从文件中加载响应内容。
  • WebSocket抓包功能
  • 抓包记录可以导出到HttpServer中，作为模板模拟接口的请求与返回
  • 抓包记录可以批量生成回放脚本实现一键回归测试。

V1.1.0(2014-05-18)

  • 优化: 优化抓包软件的界面样式，增强用户体验
  • 优化: Http Header中如果未指定Content-Type，增加可选项，用户可指定显示的字符编码
  • 新功能：支持Http/2.0协议抓包功能
  • 新功能：支持拦截指定的Http请求，并返回指定的结果
  • 新功能：支持自动设置系统代理的拦截白名单

V1.1.0

  • 升级界面UI样式，改进重访功能的界面使用体验

  • 修复抓包请求过多会内存溢出的问题

自定义证书

软件默认自带了一个根证书，正常直接使用即可，也可以用以下命令自己制作根证书，但是需要安装对应的openssl工具

  1. 创建根证书密钥文件(自己做CA) root.key
    openssl genrsa -des3 -out root.key 2048
  2. 将root.key转为pkcs8格式
    openssl pkcs8 -topk8 -inform PEM -in root.key -outform PEM -nocrypt -out root.pem
  3. 创建根证书的申请文件 root.csr
    openssl req -new -key root.key -out root.csr
  4. 创建一个自当前日期起为期十年的根证书 root.crt
    openssl x509 -req -days 900 -sha256 -signkey root.key -in root.csr -out root.crt

Windows证书安装

软件自带的默认根证书存放位置为：{用户目录}/.byteasy/fiddler/root.cert

  1. 双击证书文件，点击“安装证书”，采用默认配置一直进行下一步，最后点击完成，导入证书完成。
  2. 打开运行（win+R快捷键），输入命令“mmc”，回车确定，如下图所示
  1. 在出现的界面，选择文件的选项，在弹出的列表点击“添加/删除管理单元”：
  1. 在如下可用的管理单元的列表中找到证书，选择添加；
  1. 勾选我的用户账户，然后点击“确定”完成
  1. 添加完成证书，点击确定。
  1. 将“中间证书颁发机构》证书“下的byteasy.cn证书拖拽到“受信任的根证书颁发机构》证书“中完成安装。

系统代理的设置方法

  • mac

"网络偏好设置》高级》代理"，填入软件启动后的地址和端口即可。如下图

也可以用打开终端，用以下命令快速切换代理设置

# 打开系统代理
sudo networksetup -setwebproxy "Wi-Fi" 127.0.0.1 8090
sudo networksetup -setwebproxystate "Wi-Fi" on
sudo networksetup -setsecurewebproxy "Wi-Fi" 127.0.0.1 8090
sudo networksetup -setsecurewebproxystate "Wi-Fi" on
# 关闭系统代理
sudo networksetup -setwebproxystate "Wi-Fi" off
sudo networksetup -setsecurewebproxystate "Wi-Fi" off
  • windows

"控制面板》网络和共享中心》Internet选项》》连接》局域网设置》代理服务器"，勾选“为LAN使用代理服务器”并填写地址和端口即可。

  • ubuntu

“设置》网络代理》手动”，填入软件的地址及端口即可。
